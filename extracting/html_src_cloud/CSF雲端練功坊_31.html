下列哪些為使用者所面對的雲端安全問題？(複選題)
A　退租後，雲端資料是否確實刪除
B　確認使用雲端服務的電腦是否安全
C　與其他人共享資源，私人的資源是否會被共享
D　是否有存取記錄可供查閱與管控
正確解答為：ABCD　您的作答值為：ABCD　答對了。TNO：U0080046
題目解析：
使用者在使用雲端服務時，需要注意雲端服務是否有提供存取記錄可供查閱與管控，亦需使用安全的電腦操作雲端服務，以確保登入帳號、密碼等重要資料不會遭受第三方竊取。另外使用者所使用的雲端服務時，有可能是由許多不同的使用者共享同一個環境，因此雲端服務供應商是否能有效的隔離不同使用者，以避免彼此之間造成干擾，或是私人的資源被共享，而造成權利受損，也是相當重要的問題。而在退租雲端服務時，亦需先確認放於雲端之資料是否已全數刪除完畢，另外在使用者退租後，如何確保曾經儲存過的資料會被徹底銷毀，避免資料外洩，仍是容易受到質疑的雲端安全問題。
2
下列哪一項不是虛擬化雲端環境興起後造成更多的安全問題？
A　更多系統漏洞
B　更容易利用相同環境來測試並找出平台弱點
C　更多的垃圾郵件及惡意程式
D　更多來自殭屍網路（Botnet）的攻擊與密碼破解
正確解答為：C　您的作答值為：C　答對了。TNO：U0080047
題目解析：
虛擬化需要在硬體上或原作業系統上建置虛擬機監督器，將硬體資源抽象化，因此除了作業系統的漏洞外，亦需要注意虛擬機監督器的漏洞，另外也由於在相同虛擬化環境下建置虛擬主機，更容易利用相同環境來測試並找出平台弱點，並進行虛擬主機間的攻擊。除此之外，虛擬化雲端環境興起使得伺服器的建置更加容易，也讓攻擊者更容易建置殭屍網路，透過分散在各地受到操控的殭屍電腦，協助攻擊者發動各種攻擊與執行各種需要大量運算能力的任務，包含了發動DDoS攻擊與分散式密碼破解運算等。垃圾郵件及惡意程式的主要攻擊來源與對象並非虛擬化雲端環境，因此與虛擬化雲端環境並無直接關聯。

關於遠端管理的安全，下列敘述哪一項錯誤？
A　協定（Protocol）的設計缺陷造成安全風險
B　缺乏強健的憑證是主要的威脅
C　遠端桌面協定（RDP）最大加密強度為128位元
D　使用Telnet相較於SSH來得安全
正確解答為：D　您的作答值為：D　答對了。TNO：U0080048
題目解析：
Telnet協定屬於非加密的協定，傳輸的資料並未加密所以容易遭到竊取，因此建議改用較安全的SSH加密網路傳輸協定，可在不安全的網路中提供安全的傳輸環境。

下列哪些可降低雲端的安全威脅？(複選題)
A　使用Two-factor認證，或是動態共用密鑰來降低認證威脅
B　透過瀏覽器使用雲端服務時，選擇SSL 3.0，之後才是TLS 1.3
C　微軟的遠程桌面工具使用高強度加密方式，並要求伺服器認證；使用VNC時，在SSH或TLS tunnel上使用
D　SSH工具使用RSA密鑰進行認證
正確解答為：ACD　您的作答值為：ABCD　回答錯誤了。TNO：U0080049
題目解析：
SSL不具備客戶端身分辨識功能，且SSL 3.0存在設計缺陷，RFC 7568標準在2015年已棄用了SSL 3.0，目前較安全的協定為TLS 1.3。

下列哪些為雲端安全聯盟（Cloud Security Alliance, CSA）所指出雲端運算可能遭遇的安全威脅？(複選題)
A　使用者隱私無法保障
B　濫用和惡意使用雲端服務
C　不安全的介面與應用程式介面（APIs）
D　資料外洩
正確解答為：BCD　您的作答值為：ACD　回答錯誤了。TNO：U0080050
題目解析：
雲端安全聯盟（Cloud Security Alliance, CSA）所發表的「The Treacherous 12–Cloud Computing Top Threats」的報告中，列出12類雲端核心威脅有：Data Breaches（資料外洩）、Insufficient Identity, Credential, and Access Management（身分、憑證和存取管理不足）、Insecure Interfaces and Application Programming Interfaces （不安全的界面和APIs）、System Vulnerabilities（系統漏洞）、Account Hijacking（帳戶劫持）、Malicious Insiders（惡意的內部人員）、Advanced Persistent Threats（APTs，進階且持續性的威脅）、Data Loss（資料遺失）、Insufficient Due Diligence（審慎評鑑不足）、Abuse and Nefarious use of Cloud Services（濫用和惡意使用雲端服務）、Denial of Service（DoS，拒絕服務）、Shared Technology Vulnerabilities（共享技術漏洞）。
